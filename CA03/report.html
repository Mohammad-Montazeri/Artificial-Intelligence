7 - Exploring different methods to deal with missing values¶

  1. Deletion of Rows or Columns:

    • Pros: Simple and straightforward approach, avoiding imputation of potentially inaccurate values.
    • Cons: Leads to loss of information, especially if the column contains crucial data. May reduce dataset size, impacting model performance.

  2. Imputation with Statistics:

    • Mean/Median/Mode Imputation:
      • Pros: Preserves overall dataset structure; simple to implement.
      • Cons: Ignores potential feature relationships; sensitive to outliers, especially when using the mean.
    • Imputation with Machine Learning Models:
      • Pros: Considers relationships between features; provides a sophisticated way to fill missing values.
      • Cons: Requires additional computational resources; performance depends on the chosen model.

  3. Interpolation:

    • Pros: Suitable for time-series data; preserves trends in the dataset.
    • Cons: May not work well for non-linear or irregularly sampled data.

  4. Predictive Modeling:

    • Pros: Utilizes existing information to predict missing values.
    • Cons: Complexity depends on the chosen model; overfitting can be an issue.

  5. Multiple Imputation:

    • Pros: Provides a range of plausible values for missing data; accounts for uncertainty in imputation.
    • Cons: Requires multiple imputations, which can be computationally intensive.

9 - What are normalizing and standardizing and why do we need them?¶

In numerical features, normalizing or standardizing is performed to bring the values of different features to a similar scale. This process helps prevent certain features from dominating others, especially in machine learning algorithms that are sensitive to the scale of input features. Normalizing typically scales the data to a range of 0 to 1, while standardizing transforms the data to have a mean of 0 and a standard deviation of 1. These techniques make it easier for models to learn patterns from the data and improve the convergence and performance of various algorithms.
In this project, we'd better use one of these methods because the data of some parameters are noticeably larger than others. This gets more crucial for some algorithms with more sensitivity to the scale, such as K-Nearest-Neighbors (KNN).

10 - Preprocessing techniques for categorical features¶

For using categorical features, which are typically stored as strings or objects in a dataset, several preprocessing techniques are useful during model training. Some of these techniques include:

  1. Label Encoding: Convert categorical labels into numerical values. Suitable for ordinal categorical data, where the order matters.

  2. One-Hot Encoding: Represent each category as a binary vector, creating binary columns for each category. Suitable for nominal categorical data, where there is no inherent order among categories.

  3. Embedding: Map categorical values to continuous vectors of fixed dimensions. Commonly employed in deep learning models for handling categorical data with large cardinality.

  4. Frequency Encoding: Encode categorical labels based on their frequency or occurrence in the dataset. Can be useful when the frequency of categories carries relevant information.

  5. Target Encoding (Mean Encoding): Encode categorical labels based on the mean of the target variable for each category. Effective when there is a correlation between the categorical feature and the target variable.

  6. Handling Missing Values: Address missing values in categorical features through techniques like imputation or creating a separate category for missing values. Essential for models that cannot handle missing data.

It's not necessary for all categorical data to undergo these preprocessing techniques. The choice depends on the nature of the categorical features, the machine learning algorithm being used, and the specific requirements of the problem. Here we use One-Hot Encoding method to pre-process the categorical features Education and Marital_Status.

13 - What's the 3rd set which is called validation?¶

Sometimes, in addition to the two sets of train and test, there is a third set known as the validation or holdout set. The validation set is used during the training phase to fine-tune the model's hyperparameters and assess its performance before final evaluation on the test set. It helps prevent overfitting by providing a dataset independent of the training set for model validation. 14 - What is K-Fold Cross-Validation?¶

K-Fold Cross-Validation is a popular technique in machine learning for assessing the performance and generalizability of a model. The basic idea is to partition the original dataset into K subsets or folds. The model is trained on K-1 of these folds and validated on the remaining one. This process is repeated K times, with each fold serving as the validation set exactly once. The performance scores from each iteration are then averaged to obtain a more robust and reliable estimate of the model's performance. Here are the key steps of K-Fold Cross-Validation:

  1. Dataset Splitting: The original dataset is divided into K subsets or folds.

  2. Model Training and Validation: The model is trained K times, each time using K-1 folds for training and the remaining fold for validation.

  3. Performance Evaluation: The performance metric (such as accuracy, precision, recall, etc.) is recorded for each iteration.

  4. Average Performance: The average performance across all K iterations is calculated, providing a more stable and representative estimate of the model's performance.

Common choices for the value of K include 5 or 10, but the choice depends on factors such as the size of the dataset and computational resources available.

Phase One: Linear Regression¶

In this phase of the project, we will focus on building a first-degree linear regression model without using any pre-built library, except for NumPy. Note that, in this phase, the use of any external libraries (except for NumPy) is strictly prohibited.

Main form of simple linear regression function: $$f(x) = \alpha x + \beta$$

here we want to find the bias ($\alpha$) and slope($\beta$) by minimizing the derivation of the Residual Sum of Squares (RSS) function:

  • step 1: Compute RSS of the training data

$$ RSS = \Sigma (y_i - (\hat{\beta} + \hat{\alpha} * x_i) )^2 $$

  • step 2: Compute the derivatives of the RSS function in terms of $\alpha$ and $\beta$, and set them equal to 0 to find the desired parameters

$$ \frac{\partial RSS}{\partial \beta} = \Sigma (-f(x_i) + \hat{\beta} + \hat{\alpha} * x_i) = 0$$ $$ \to \beta = \hat{y} - \hat{\alpha} \hat{x} \to (1)$$

$$ \frac{\partial RSS}{\partial \alpha} = \Sigma (-2 x_i y_i + 2 \hat{\beta} x_i + 2\hat{\alpha} x_i ^ 2) = 0 \to (2)$$

$$ (1) , (2) \to \hat{\alpha} = \frac{\Sigma{(x_i - \hat{x})(y_i - \hat{y})}}{\Sigma{(x_i - \hat{x})^2}} $$ $$ \hat{\beta} = y - \hat{a} x$$

Based on the above formula, implement the function below to compute the parameters of a simple linear regression

In [ ]:
import numpy as np

def simple_linear_regression(input_feature, output):

    x_hat = np.mean(input_feature)
    y_hat = np.mean(output)
    delta_x = input_feature - x_hat
    delta_y = output - y_hat

    slope = np.sum(delta_x*delta_y) / np.sum(delta_x**2)
    bias = y_hat - slope * x_hat
    return slope, bias

def get_root_mean_square_error(predicted_values, outputs):
    RMSE = np.sqrt(np.mean((predicted_values - outputs)**2))
    return RMSE

The RMSE has no bound, thus it becomes challenging to determine whether a particular RMSE value is considered good or bad without any reference point. Instead, we use R2 score. The R2 score is calculated by comparing the sum of the squared differences between the actual and predicted values of the dependent variable to the total sum of squared differences between the actual and mean values of the independent variable. The R2 score is formulated as below:

$$R^2 = 1 - \frac{SSres}{SStot} = 1 - \frac{\sum_{i=1}^{n} (y_{i,true} - y_{i,pred})^2}{\sum_{i=1}^{n} (y_{i,true} - \bar{y}_{true})^2} $$

def get_r2_score(predicted_values, outputs):
    dependent = np.sum((outputs - predicted_values)**2)
    independent = np.sum((outputs - np.mean(outputs))**2)
    R2 = 1 - dependent/independent
    return R2

17 - Exploring methods of evaluation¶

  1. RSS (Residual Sum of Squares):

    • Definition: RSS is the sum of the squared differences between the actual and predicted values.
    • Purpose: Measures the overall fit of the model by quantifying the total squared residuals.
    • Interpretation: A lower RSS indicates a better fit, but it is not normalized.

  2. MSE (Mean Squared Error):

    • Definition: MSE is the average of the squared differences between the actual and predicted values.
    • Purpose: Similar to RSS, but normalized by the number of data points, providing a mean measure of the squared errors.
    • Interpretation: Lower MSE values indicate better model performance.

  3. RMSE (Root Mean Squared Error):

    • Definition: RMSE is the square root of the MSE.
    • Purpose: Provides an interpretable measure in the original units of the target variable, addressing the issue of squared units.
    • Interpretation: Lower RMSE values suggest better model accuracy.

  4. R-squared (R2 Score):

    • Definition: R2 measures the proportion of the variance in the dependent variable explained by the independent variables.
    • Purpose: Indicates the goodness of fit, ranging from 0 to 1.
    • Interpretation: Higher R2 values (closer to 1) imply a better model fit, with 0 indicating no explanatory power.

When reporting on these metrics, it is essential to consider the specific context of your problem and choose the metrics that align with your evaluation goals. Lower values of RSS, MSE, and RMSE are desirable, while higher values of R2 indicate better model performance.

18 - Evaluating the results¶

A large R-squared (R2) score is generally considered good when evaluating a regression model. The R2 score, also known as the coefficient of determination, measures the proportion of the variance in the dependent variable that is predictable from the independent variables. It ranges from 0 to 1, where:

  • 0 indicates that the model does not explain any variability in the dependent variable.
  • 1 indicates that the model perfectly explains the variability in the dependent variable.

For RMSE:

  • A lower RMSE indicates better model performance.
  • It is in the same unit as the dependent variable, making it interpretable in the context of the problem.

Phase Two: Multiple Regression¶

Multiple regression is a statistical technique that aims to model the relationship between a dependent variable and two or more independent variables.

Multiple regression with n independent variables is expressed as follows:

$$f(x) = \beta _{0} + \beta_{1} x_{1} + \beta_{2} x_{2} + \beta_{3} x_{3} + \beta_{4} x_{4} + ... + \beta_{n} x_{n} + c $$

To optimize the model for accurate predictions, multiple regression commonly employs iterative algorithms such as gradient descent.

The main goal of the optimization process is to make our predictions as close as possible to the actual values. We measure the prediction error using a cost function, usually denoted as $J(\beta)$.

$$ J(\beta)= \frac {1}{2m} Σ_{i=0}^{m-1}(y_i - (\hat \beta _{0} + \hat \beta_{1} x_{1} + \hat \beta_{2} x_{2} + \hat \beta_{3} x_{3} + \hat \beta_{4} x_{4} + ... + \hat \beta_{n} x_{n}) )^2 $$

Gradient descent iteratively adjusts the coefficients $(\beta_i)$ to minimize the cost function. The update rule for each coefficient is:

$$\beta_{i} = \beta _ {i} - \alpha \frac {∂J(\beta)}{∂\beta_{i}}$$

$$ \frac {∂J(\beta)}{∂\beta_{i}} = \frac {1}{m}Σ_{j=0}^{m-1}(y_j - (\hat \beta _{0} + \hat \beta_{1} x_{j1} + \hat \beta_{2} x_{j2} + \hat \beta_{3} x_{j3} + \hat \beta_{4} x_{j4} + ... + \hat \beta_{n} x_{jn})) x_{ji} $$

def predict_output(feature_matrix, weights, bias):
    # assume feature_matrix is a numpy matrix containing the features as columns and weights is a corresponding numpy array
    # create the predictions vector by using np.dot()
    predictions = np.dot(feature_matrix, weights) + bias
    return predictions

Computing the Derivative¶

As we saw, the cost function is the sum over the data points of the squared difference between an observed output and a predicted output.

Since the derivative of a sum is the sum of the derivatives, we can compute the derivative for a single data point and then sum over data points. We can write the squared difference between the observed output and predicted output for a single point as follows:

$$ (output - (const* w _{0} + [feature_1] * w_{1} + ...+ [feature_n] * w_{n} ))^2 $$

With n feautures and a const , So the derivative will be :

$$ 2 * (output - (const* w _{0} + [feature_1] * w_{1} + ...+ [feature_n] * w_{n} )) $$

The term inside the paranethesis is just the error (difference between prediction and output). So we can re-write this as:

$$2 * error*[feature_i] $$

That is, the derivative for the weight for feature i is the sum (over data points) of 2 times the product of the error and the feature itself. In the case of the constant then this is just twice the sum of the errors!

Recall that twice the sum of the product of two vectors is just twice the dot product of the two vectors. Therefore the derivative for the weight for feature_i is just two times the dot product between the values of feature_i and the current errors.

With this in mind, complete the following derivative function which computes the derivative of the weight given the value of the feature (over all data points) and the errors (over all data points).

In [ ]:
def feature_derivative(errors, feature):
    derivative = 2 * np.dot(errors, feature)
    return derivative

Gradient Descent¶

Now we will write a function that performs a gradient descent. The basic premise is simple. Given a starting point we update the current weights by moving in the negative gradient direction. Recall that the gradient is the direction of increase and therefore the negative gradient is the direction of decrease and we're trying to minimize a cost function.

The amount by which we move in the negative gradient direction is called the 'step size'. We stop when we are 'sufficiently close' to the optimum. We define this by requiring that the magnitude (length) of the gradient vector to be smaller than a fixed 'tolerance'.

With this in mind, complete the following gradient descent function below using your derivative function above. For each step in the gradient descent we update the weight for each feature befofe computing our stopping criteria.

def regression_gradient_descent(feature_matrix, outputs, initial_weights, bias, step_size, tolerance):
    new_w = initial_weights
    new_b = bias
    iteration = 0
    while iteration <= 10000:
        prediction = predict_output(feature_matrix, new_w, new_b)
        err = prediction - outputs
        gradient_w = feature_derivative(feature_matrix.T, err)
        gradient_b = np.sum(err)
        new_w -= step_size * gradient_w
        new_b -= step_size * gradient_b
        iteration += 1        
        if np.linalg.norm(gradient_w) < tolerance:
            break

    return new_w, new_b

Answers to the questions in the code cell above¶

Would selecting different initial weights make any difference? Explain your answer.¶

Yes, the choice of initial weights in the gradient descent algorithm can influence the convergence and performance of the algorithm. The impact of different initial weights is more pronounced in non-convex optimization problems, and it's often associated with the presence of local minima, saddle points, and other complex geometries in the loss landscape.

Here are some considerations regarding the choice of initial weights:

  1. Convergence Speed: The choice of initial weights can affect how quickly the algorithm converges to a solution. If the initial weights are close to an optimal solution, the convergence may be faster. On the other hand, choosing very distant initial weights might slow down convergence.

  2. Avoiding Local Minima: In non-convex optimization problems, different initial weights may lead the algorithm to different local minima. It's possible that a particular choice of initial weights gets stuck in a suboptimal solution, while another choice finds a better solution.

  3. Robustness: A robust optimization algorithm should be able to handle various initializations and still find a reasonable solution. If the algorithm is sensitive to the choice of initial weights, it may not be as robust.

  4. Random Initialization: To mitigate the impact of the choice of initial weights, it's common to use random initialization. This involves setting the initial weights to random values, which can help the algorithm explore different regions of the optimization landscape.

  5. Saddle Points: In some cases, poor choices of initial weights can lead the algorithm to saddle points rather than minima. Techniques such as random initialization and using adaptive learning rates can help address this issue.

In practice, it's common to experiment with different initializations and choose the one that yields satisfactory results. Random initialization, combined with techniques like learning rate schedules and adaptive optimization methods (e.g., Adam, RMSprop), is often used to enhance the robustness of gradient descent algorithms.

Why are the features normalized?¶

Normalizing features is a crucial preprocessing step in machine learning that involves scaling the numerical values of features to a common range. This practice is particularly beneficial for optimization-based algorithms like gradient descent. One key advantage of normalization is the improved convergence of optimization algorithms. When features have different scales, the learning rates for each feature may need adjustment, and normalizing ensures a balanced and consistent learning process. Additionally, normalization helps prevent numerical instability issues that might arise when dealing with features with large values. By equalizing the influence of features, normalization ensures that all features contribute more uniformly to the learning process, facilitating fair comparisons and meaningful interpretation of model coefficients. Regularization techniques, which penalize large coefficients, also assume similar scales among features, making normalization essential for effective regularization. In distance-based algorithms, such as k-nearest neighbors or support vector machines, normalization is crucial to ensure fair comparisons. While not all algorithms require feature normalization, it is a good practice for many optimization-based models, contributing to stability, efficiency, and better overall model performance.

Explaining the results¶

RMSE (Root Mean Squared Error): The RMSE is a measure of the average magnitude of the errors between predicted and actual values, which is 5.1010 here. This value represents the square root of the average squared differences between the predicted and true values. A lower RMSE indicates better model performance, as it signifies that the model's predictions are, on average, closer to the actual values. In summary, an RMSE of 5.1010 suggests that, on average, your model's predictions have an error of approximately 5.1 units.

R2 Score (Coefficient of Determination): The R2 score quantifies the proportion of the variance in the dependent variable (target) that is explained by the independent variables (features) in the model. A value of 1 indicates a perfect fit, while a value of 0 suggests that the model's predictions are no better than simply predicting the mean of the target. In this case, the R2 score is 0.5590. This means that approximately 55.9% of the variance in the target variable is explained by the model. So, the overall performance can be considered moderate.
